  </behavior>
  <action>
    Create `src/content_extractor/analysis.py` following the pattern established in `vision.py`:

    1. Define `AnalysisError(Exception)` for LLM failures.

    2. Define a module-level `_ANALYSIS_PROMPT` string that instructs Claude to analyze text and return JSON with exactly these fields:
       - `topics`: list of 3-7 main themes/subjects discussed
       - `viewpoints`: list of core arguments or perspectives presented
       - `sentiment`: object with `overall` ("positive"/"negative"/"neutral"/"mixed") and `confidence` (0.0-1.0)
       - `takeaways`: list of actionable insights or key lessons
       The prompt should say: "Return ONLY valid JSON, no markdown fences or extra text."
       The prompt should handle both Chinese and English content.

    3. Implement `analyze_content(raw_text: str, *, content_id: str, content_type: str, config: ExtractorConfig | None = None) -> AnalysisResult`:
       - If `raw_text` is empty or whitespace-only, return a fallback `AnalysisResult` with empty fields (no LLM call).
       - Create client via `create_claude_client(config)`.
       - Call `client.messages.create()` with the analysis prompt + the raw_text as user content.
       - Parse JSON response with `orjson.loads()`.
       - Map parsed dict to `AnalysisResult` Pydantic model (convert sentiment dict to `SentimentResult`).
       - On JSON parse failure: log warning, return fallback `AnalysisResult` with empty fields.
       - On API exception: wrap in `AnalysisError` and re-raise.

    4. Create `tests/test_analysis.py`:
       - Mock `create_claude_client` to return a mock client.
       - Mock `client.messages.create()` to return canned JSON responses.
       - Test all 6 behaviors listed above.
       - Use `unittest.mock.patch` targeting `content_extractor.analysis.create_claude_client`.
