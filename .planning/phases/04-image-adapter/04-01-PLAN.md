  <action>
    Create `src/content_extractor/vision.py` with two public functions:

    1. `preprocess_image(image_path: Path) -> tuple[str, str]`:
       - Open with `PIL.Image.open()`
       - If max(width, height) > 1568, resize proportionally using `Image.LANCZOS`
       - Determine output format: keep original if JPEG/PNG/GIF/WEBP, else convert to JPEG
       - For JPEG output, convert RGBA/P mode images to RGB first (JPEG does not support alpha)
       - Encode to bytes via `io.BytesIO`, then `base64.standard_b64encode`
       - Return (base64_string, media_type_string like "image/jpeg")

    2. `describe_image(base64_data: str, media_type: str, config: ExtractorConfig) -> VisionResponse`:
       - Create Anthropic client. For now, use `anthropic.Anthropic()` which reads ANTHROPIC_API_KEY env var. Phase 3 will provide a proper client factory — this is the minimal viable approach that works with both env var and future LLM module.
       - Build messages with image content block (type=image, source=base64) + text prompt
       - Prompt must explicitly request JSON with `ocr_text`, `visual_description`, `confidence` fields
       - Prompt must instruct: transcribe ALL visible text including Chinese characters in original language
       - Prompt must instruct: note text position context (top, bottom, overlay)
       - Use `config.claude_model`, `config.claude_max_tokens`, `config.claude_temperature`
       - Parse response text with `orjson.loads()` into `VisionResponse` Pydantic model
       - On JSON parse failure: create VisionResponse with raw text as visual_description, ocr_text="", confidence=0.0
       - On API error: raise `ImageExtractionError` with clear message including the original error

    3. Define `VisionResponse(BaseModel, frozen=True)`: ocr_text: str, visual_description: str, confidence: float

    4. Define `ImageExtractionError(Exception)` for image-specific failures

    Write tests in `tests/test_image_extractor.py` using mocked Anthropic client (unittest.mock.patch). Create test fixtures in `tests/conftest.py`:
    - `tmp_image_dir` pytest fixture that creates a temp dir with a small test JPEG (use Pillow to create a 100x100 red image)
    - `mock_vision_response` fixture returning a valid JSON string with ocr_text/visual_description/confidence

    Tests must mock `anthropic.Anthropic` — never make real API calls.
