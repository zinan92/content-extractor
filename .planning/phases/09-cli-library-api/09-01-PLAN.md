  <action>
    Create `src/content_extractor/cli.py`:

    1. Create `app = typer.Typer(name="content-extractor", help="Turn raw multimedia content into structured text.")`.

    2. `extract` command (CLI-01):
       - Argument: `path: Path` (required, must be existing directory)
       - Options: `--whisper-model` (str, default "turbo"), `--force` (bool flag, default False)
       - Validate path exists and is a directory; if not, print error via `rich.console.Console(stderr=True)` and `raise typer.Exit(code=1)`
       - Build `ExtractorConfig(whisper_model=whisper_model, force_reprocess=force)`
       - Call `extract_content(path, config)`
       - If result is None: print "[yellow]Already extracted. Use --force to reprocess.[/yellow]" via Rich Console and exit 0
       - If result returned: print summary — content_id, content_type, word count from quality.word_count, processing time from quality.processing_time_seconds
       - On exception: print error to stderr Console, `raise typer.Exit(code=1)`

    3. `extract_batch` command (CLI-02, CLI-03, CLI-04):
       - IMPORTANT: Do NOT call `extract_batch()` from extract.py directly because it does not support progress callbacks. Instead, reimplement the scan + per-item loop inside the CLI command to integrate Rich progress.
       - Argument: `path: Path` (required, must be existing directory)
       - Options: `--whisper-model` (str, default "turbo"), `--force` (bool flag, default False)
       - Validate path exists and is directory
       - Scan for content_item.json files: `sorted(p.parent for p in path.rglob("content_item.json"))`
       - Use `rich.progress.Progress` with columns: SpinnerColumn, TextColumn for item name, BarColumn, TaskProgressColumn, TimeElapsedColumn
       - For each content_dir in progress loop:
         - Update progress description with directory name
         - Try `extract_content(content_dir, config)` — track succeeded/skipped/failed counts
         - On exception: append to errors list, continue (per D-07)
       - After loop, print summary line: "Processed X items: Y succeeded, Z skipped, W failed"
       - If failures > 0 (CLI-04): print Rich Table with columns "Content Directory" and "Error", one row per failure. Use `Console(stderr=True)` for the error table.
       - Exit code 1 if any failures, 0 otherwise

    4. `def main(): app()` — entry point referenced by pyproject.toml

    Create `tests/test_cli.py`:
    - Use `typer.testing.CliRunner` to invoke commands
    - Mock `extract_content` from `content_extractor.extract` to avoid real extraction
    - Use `tmp_path` fixture to create fake content_item.json directories
    - Test all behaviors listed above
    - For progress bar test: verify output contains progress-related text (item counts) or mock Progress
