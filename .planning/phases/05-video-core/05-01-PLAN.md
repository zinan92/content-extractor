    - probe_audio_stream returns AudioProbeResult with has_audio=False for a video with no audio stream
    - probe_audio_stream raises FFmpegError when ffprobe binary is not found
    - probe_audio_stream raises FFmpegError when input file does not exist
    - extract_audio produces a WAV file at the target path with correct format (pcm_s16le, 16kHz, mono)
    - extract_audio raises FFmpegError when input has no audio stream (detected via probe)
    - extract_audio raises FFmpegError when ffmpeg returns non-zero exit code
    - extract_audio raises FFmpegError when output file is 0 bytes after extraction
    - extract_audio uses a temporary file and renames atomically on success
  </behavior>
  <action>
    Create `src/content_extractor/video/__init__.py` (empty).

    Create `src/content_extractor/video/ffmpeg.py` with:

    1. `AudioProbeResult` — frozen dataclass with fields: `has_audio: bool`, `codec: str | None`, `sample_rate: int | None`, `duration_seconds: float | None`.

    2. `FFmpegError(Exception)` — custom exception for all FFmpeg/ffprobe failures.

    3. `probe_audio_stream(video_path: Path) -> AudioProbeResult` — runs `ffprobe -v error -select_streams a:0 -show_entries stream=codec_name,sample_rate,duration -of json <video_path>` via `subprocess.run(capture_output=True, timeout=30)`. Parse JSON output. If no audio streams in output, return `AudioProbeResult(has_audio=False, ...)`. If `FileNotFoundError` from subprocess (ffprobe not installed), raise `FFmpegError("ffprobe not found. Install FFmpeg: brew install ffmpeg")`. If video file does not exist, raise `FFmpegError` with path.

    4. `extract_audio(video_path: Path, output_path: Path) -> Path` — first calls `probe_audio_stream` to verify audio exists. If no audio, raise `FFmpegError(f"No audio stream in {video_path}")`. Then runs `ffmpeg -i <video> -vn -acodec pcm_s16le -ar 16000 -ac 1 -y -f wav <tmp_path>` where `tmp_path = output_path.with_suffix('.wav.tmp')`. Capture stderr for logging. On non-zero return code, raise `FFmpegError` including stderr text. On success, verify tmp file size > 0 bytes, then `tmp_path.rename(output_path)` for atomic write. Return `output_path`.

    Write tests in `tests/test_ffmpeg.py`:
    - Mock `subprocess.run` to simulate ffprobe JSON output (with audio, without audio, not found error, non-existent file).
    - Mock `subprocess.run` to simulate ffmpeg success (create a dummy tmp file in the mocked call), failure (non-zero returncode), and 0-byte output.
    - Use `tmp_path` pytest fixture for temp directories.
    - Do NOT call real ffmpeg/ffprobe in unit tests.
  </action>

    Create `src/content_extractor/video/transcribe.py` with:

    1. `TranscriptionError(Exception)` — custom exception.

    2. Module-level `_model_cache: dict[str, WhisperModel] = {}` — cache loaded models by model name to avoid reloading on every call in batch mode. This is the ONE mutable module-level state, justified by the ~5 second model load time.

    3. `_get_model(model_name: str) -> WhisperModel` — returns cached model or loads new one. Wraps `WhisperModel(model_size_or_path=model_name, device="auto", compute_type="int8")` in try/except, raises `TranscriptionError` on failure.

    4. `transcribe_audio(audio_path: Path, *, whisper_model: str = "turbo", language: str = "zh") -> tuple[TranscriptSegment, ...]`:
       - Get model via `_get_model(whisper_model)`
       - Call `model.transcribe(str(audio_path), language=language, initial_prompt="以下是普通话的句子。", condition_on_previous_text=False, beam_size=5, vad_filter=True, vad_parameters={"min_silence_duration_ms": 500})`
       - Iterate over segments generator. For each segment:
         - Skip if `segment.no_speech_prob > 0.6`
         - Calculate confidence: `math.exp(segment.avg_logprob)`, clamp to [0.0, 1.0]
         - Create `TranscriptSegment(text=segment.text.strip(), start=segment.start, end=segment.end, confidence=confidence)`
       - Return as tuple
