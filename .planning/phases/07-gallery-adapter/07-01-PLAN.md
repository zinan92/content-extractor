    - Test 2: Gallery with 7 images makes 2 batches (5 + 2), still produces correct results
    - Test 3: One image fails (FileNotFoundError) -> fallback MediaDescription(confidence=0.0), gallery still succeeds
    - Test 4: All images fail -> ExtractionResult with empty narrative, all MediaDescriptions at confidence=0.0
    - Test 5: Narrative synthesis combines per-image descriptions into coherent text
    - Test 6: Empty gallery (no images found) raises ImageExtractionError
    - Test 7: Quality metadata has correct avg confidence, language, word count, processing time
  </behavior>
  <action>
    Replace the stub in `src/content_extractor/adapters/gallery.py` with full GalleryExtractor.

    **Image discovery** (per GLRY-01): Copy the pattern from ImageExtractor exactly:
    1. Load ContentItem via `load_content_item(content_dir)`
    2. Filter `item.media_files` by `_IMAGE_EXTENSIONS`
    3. Fallback: glob `media/` directory for image files
    4. Raise `ImageExtractionError` if no images found

    **Batched extraction** (per GLRY-02): Process images in batches of `_BATCH_SIZE = 5`:
    - For each batch, iterate images and call `preprocess_image()` + `describe_image()` per image
    - Per-image try/except: on failure, append fallback `MediaDescription(file_path=..., description="", ocr_text="", confidence=0.0)` and log warning
    - Note: "batching" means sequential processing in groups with a brief `time.sleep(1.0)` between batches to respect rate limits, NOT multi-image-per-API-call (Claude vision API takes one image per call)

    **Narrative synthesis** (per GLRY-03): After all images are described:
    1. Collect descriptions that have non-empty `visual_description` or `ocr_text`
    2. If no successful descriptions, set `narrative = ""` and skip LLM call
    3. Otherwise, call `create_claude_client(config).messages.create()` with a text-only prompt:
       - System: "You are a content analyst. Synthesize individual image descriptions from a gallery post into a coherent narrative."
       - User: numbered list of per-image descriptions (format: "Image 1: {visual_description}. Text: {ocr_text}")
       - Request: "Write a coherent narrative (2-4 paragraphs) describing this gallery as a whole. Preserve any Chinese text references. Focus on the story arc and key information across all images."
    4. Parse response text as the narrative string

    **Return ExtractionResult**: content_type="gallery", raw_text=narrative, media_descriptions=tuple(all descriptions), quality with avg confidence, detected language from narrative, word count, processing time.

    **Tests** in `tests/test_gallery_extractor.py`:
    - Mock `preprocess_image` and `describe_image` from vision module (same pattern as test_image_extractor.py)
    - Mock `create_claude_client` for the narrative synthesis call
    - Use `tmp_image_dir` fixture from conftest.py for real image file tests
    - Use `mock_vision_response_json` fixture pattern
  </action>
